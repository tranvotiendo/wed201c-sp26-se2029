Phần 1

Khái niệm File (Tệp tin)

Đơn vị logic của thông tin, tồn tại bền vững trên bộ nhớ thứ cấp.

persistent (Bền vững)

Thông tin trong file phải tồn tại độc lập với việc tiến trình tạo hay kết thúc.

Naming (Đặt tên)

Tên file là chuỗi ký tự. Phần mở rộng (extension) gợi ý nội dung (.c, .txt, .bat).

Structure (Cấu trúc)

Byte sequence (không cấu trúc), Record sequence (bản ghi), hoặc Tree (cây).

Phân loại File (File Types)

Regular

Chứa dữ liệu người dùng (ASCII hoặc Binary).

Directory

File hệ thống duy trì cấu trúc thư mục.

Special

Character special (thiết bị tuần tự) & Block special (thiết bị khối).

Phương thức Truy cập & Thuộc tính

Access methods

  • • Sequential: Tuần tự.
  • • Random: Ngẫu nhiên (ổ đĩa).

Metadata

  • • Owner, Access rights.
  • • Timestamps, Size.

System Calls

open() read() write() seek()

Thư mục (Directories)

Ánh xạ tên file ASCII sang thông tin dữ liệu vật lý.

Single-level System

Một thư mục duy nhất chứa tất cả file. Đơn giản nhưng không thể trùng tên và khó mở rộng.

Hierarchical System

Thư mục chứa thư mục con, tạo cấu trúc cây. Giúp nhóm file hiệu quả và cá nhân hóa root directory.

/ (root)
├── home/
├── john/ (Working Directory)
└── program.c
Absolute Path

Bắt đầu từ gốc:
/home/students/john/pro.c

Relative Path

Từ thư mục hiện tại:
john/program.c

Special Dirs

. (Hiện tại)
.. (Thư mục cha)

FAT (File Allocation Table)

Bảng con trỏ trong RAM. Truy cập ngẫu nhiên nhanh hơn, nhưng FAT chiếm bộ nhớ.

Journaling (JFS) & Log-Structured (LFS)

LFS: Coi toàn bộ đĩa như một log, ghi theo phân đoạn lớn để tăng hiệu suất. Cần luồng "cleaner" dọn dẹp.

JFS: Ghi nhật ký thao tác dự kiến để khôi phục sau sự cố (NTFS, ext3). Đảm bảo tính nhất quán.

Virtual File System (VFS)

Phần 4

Quản lý & Tối ưu hóa

Disk Space Mgmt

Block Size: Cân bằng giữa hiệu năng (khối lớn) và tiết kiệm không gian (khối nhỏ).

Free Space: Theo dõi bằng Linked List hoặc Bitmap (bit=1 nếu trống).

Disk Quotas: Hạn mức số tệp và khối cho mỗi người dùng.

Performance Tuning

Block Cache: Lưu khối đĩa thường dùng trong RAM (LRU algorithm).

Read-Ahead: Tải trước khối kế tiếp cho truy cập tuần tự.

Defragmentation: Sắp xếp lại file để chúng liền mạch trên đĩa.

Tính nhất quán & Sao lưu

Consistency (Tính nhất quán)

Sự cố có thể gây sai lệch i-node hoặc khối. Sử dụng fsck (Linux) hoặc scandisk (Windows) để sửa lỗi.

Backups (Sao lưu)

Physical dump (toàn bộ đĩa) và Logical dump (các file thay đổi từ mốc thời gian nhất định).

Hệ thống File mẫu (Example FS)

CD-ROM (ISO 9660)

Môi trường ghi một lần, rãnh xoắn ốc liên tục.

  • • Rock Ridge: Mở rộng cho UNIX.
  • • Joliet: Mở rộng cho Windows (Unicode).

MS-DOS System

Cấu trúc thư mục phẳng, mục cố định 32 byte.

  • • Tên tệp định dạng 8.3.
  • • Quản lý qua bảng FAT.

UNIX V7

Cấu trúc cây dạng Đồ thị có hướng không chu trình (DAG).

  • • i-node chứa metadata & pointers.
  • • Mục thư mục: Tên (14B) + i-node # (2B).
