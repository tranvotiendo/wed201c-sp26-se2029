Part 4: System Architecture

Exploring the core relationship between Android and Linux

Android: Linux on Mobile

Android is not a traditional Linux distribution, but a platform "Standing on the shoulders of Linux" with 3 core tasks:

  • Drivers: Hardware recognition (camera, screen, sensors).
  • Power Management: Optimizing mobile battery life.
  • Security: Utilizing UNIX-based permissions system.

Android 5-Layer Stack

  • 1. System Apps: Everyday apps like Dialer, Email, Camera.
  • 2. Java API Framework: Tools provided for developers.
  • 3. HAL & Runtime: The intermediate processing "Brain".
  • 4. Native Libraries: C/C++ libs for graphics and media.
  • 5. Linux Kernel: Direct hardware management layer.

Secrets of Smoothness: HAL & ART

HAL

Hardware Abstraction Layer: A standard interface helping apps talk to Drivers regardless of hardware details.

ART

Android Runtime: Uses Ahead-of-Time (AOT) compilation for faster app launches and better battery efficiency.

Sandbox

UNIX-inherited mechanism: Each app runs in its own process with a unique UID for maximum security.
