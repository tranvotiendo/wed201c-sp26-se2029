Operating System Fundamentals

Language: English / Tiếng Việt

1. Operating System Overview

Tổng quan về Hệ điều hành

1.1 Định nghĩa & Vai trò

  • Core Software / Phần mềm cốt lõi

    Manages computer hardware and software resources. (Quản lý tài nguyên phần cứng và phần mềm).

  • Intermediary / Trung gian

    Acts between user and hardware. (Đóng vai trò trung gian giữa người dùng và phần cứng).

  • Convenience / Sự tiện lợi

    Hides hardware complexity. (Ẩn đi sự phức tạp của phần cứng).

1.2 Mục tiêu (Goals)

Execute Programs (Thực thi chương trình)
Ease of Use (Dễ sử dụng)
Resource Management (Quản lý tài nguyên)
Stable Environment (Môi trường ổn định)

1.3 System Structure / Cấu trúc hệ thống

Users (Người dùng)
↓
Applications (Ứng dụng)
↓
Operating System (Hệ điều hành)
↓

Hardware (Phần cứng)

CPU, Memory, I/O Devices

Kernel Mode (Chế độ nhân)

OS runs with full control over hardware. (Hệ điều hành chạy với toàn quyền kiểm soát).

User Mode (Chế độ người dùng)

Applications run with limited access for safety. (Ứng dụng chạy với quyền hạn chế để đảm bảo an toàn).

2. The Operating System Zoo

Phân loại các hệ điều hành

🖥️

Mainframe OS

Xử lý nhiều công việc I/O cùng lúc.

  • • Batch: Chạy theo lô.
  • • Transaction: Xử lý giao dịch.
  • • Time-sharing: Đa nhiệm.
  • Ex: OS/360, Linux, UNIX.
🌐

Server OS

Phục vụ nhiều người dùng qua mạng.

  • • Chia sẻ tệp, ứng dụng, web.
  • • Chạy trên máy chủ lớn.
  • Ex: Windows Server, Solaris, Linux.
🚀

Multiprocessor OS

Hệ thống có nhiều CPU/Lõi.

  • • Increased throughput.
  • • Resource sharing.
  • • High reliability.
  • Ex: Windows, SunOS, OS/2.
💻

Personal Computer

Tập trung vào sự tiện lợi cá nhân.

  • • Word, Web, Games...
  • • single user interface.
  • Ex: Windows 11, macOS, Ubuntu.
📱

Handheld (Mobile)

Thiết bị di động màn hình nhỏ.

  • • Limited memory & CPU.
  • • Power efficiency.
  • Ex: Android, iOS, Symbian.
⚙️

Embedded OS

Tích hợp trong TV, lò vi sóng, ô tô.

  • • Fixed functions.
  • • Stored in ROM.
  • Ex: VxWorks, QNX.
⏱️

Real-Time (RTOS)

Đáp ứng thời hạn nghiêm ngặt.

  • • Hard: Tuyệt đối không trễ.
  • • Soft: Có thể trễ đôi khi.
  • Ex: eCos.

Sensor Nodes

Giám sát môi trường, tiêu thụ ít năng lượng.

Ex: TinyOS.

Smart Card OS

Giao dịch bảo mật, ATM, Credit card.

Ex: ATM, Credit cards.

3. Operating System Concepts

Các khái niệm hệ thống then chốt

Processes / Tiến trình Execution

Một chương trình đang được thực thi và nạp vào bộ nhớ.

Tài nguyên: Stack, Data, CPU Regs, PC
Cấu trúc: Hệ phân cấp (Parent/Child)

Address Space / Không gian địa chỉ Memory

Ảo giác về vùng nhớ riêng biệt (0 → Max) cho mỗi tiến trình.

Giúp tránh xung đột địa chỉ và cho phép hoán đổi (swapping) linh hoạt.

Files / Tệp tin Storage

Trừu tượng hóa dữ liệu lưu trên đĩa.

Block Special (Disk)
Character Special (Printer)
Pipes (Connecting processes)

Input/Output & Protection

I/O Management

Điều phối phần mềm/phần cứng qua I/O Subsystem và Device Drivers.

Protection

Kiểm soát truy cập, ngăn chặn can thiệp trái phép, bảo vệ mã OS.

$ Shell

Giao diện giữa người dùng và OS (Command Line hoặc GUI).

sh, bash, ksh, csh...
Shell scripts: Tệp chứa chuỗi lệnh.

4. Ontogeny Recapitulates Phylogeny

Sự phát triển lặp lại lịch sử

Early

Thời kỳ đầu

Chạy trực tiếp trên phần cứng, tập lệnh cố định, bộ nhớ cực kỳ hạn chế, không có cơ chế bảo vệ.

Mid

Cải tiến trung gian

Vi lập trình (Microprogramming) tăng tốc độ và tính linh hoạt.

Now

Hiện đại

Bộ nhớ khổng lồ (GB/TB), Bộ nhớ ảo (Virtual Memory) chạy ứng dụng lớn hơn RAM vật lý, đa nhiệm mạnh mẽ.
