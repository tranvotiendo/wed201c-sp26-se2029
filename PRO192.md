## 0. Syntax cơ bản trong Java

### 0.1. Các quy tắc cơ bản
- Mỗi câu lệnh phải kết thúc bằng dấu chấm phẩy (;)
- Code Java phân biệt chữ hoa và chữ thường (case-sensitive)
- Các khối lệnh được đặt trong cặp dấu ngoặc nhọn { }
- Có thể sử dụng khoảng trắng và xuống dòng để code dễ đọc hơn

### 0.2. Comments (Chú thích)
```java
// Đây là comment một dòng

/* Đây là comment nhiều dòng
   Có thể viết nhiều dòng
   Như thế này */

/** Đây là documentation comment 
  * Thường dùng để tạo JavaDoc
  * @author Minthep
  */
```

### 0.3. Quy tắc đặt tên
1. Tên biến:
   - Bắt đầu bằng chữ cái, $ hoặc _
   - Sử dụng camelCase: firstName, lastName
   - Không được dùng từ khóa của Java
   Ví dụ:
   ```java
   int age;
   String firstName;
   double $salary;
   boolean _isStudent;
   // Không hợp lệ: int 2ndNumber; String void;
   ```

2. Tên class:
   - Bắt đầu bằng chữ in hoa
   - Sử dụng PascalCase: HelloWorld, StudentManager
   Ví dụ:
   ```java
   public class HelloWorld { }
   public class StudentManager { }
   public class BankAccount { }
   ```

3. Tên package:
   - Tất cả viết thường
   - Các từ cách nhau bằng dấu chấm: com.example.project
   Ví dụ:
   ```java
   package com.mycompany.project;
   package org.example.utilities;
   package edu.myuniversity.courses;
   ```

4. Tên hằng số:
   - Tất cả viết hoa
   - Các từ cách nhau bằng dấu gạch dưới
   Ví dụ:
   ```java
   public static final int MAX_SIZE = 100;
   public static final String DATABASE_URL = "jdbc:mysql://localhost/mydb";
   ```

5. Tên phương thức:
   - Bắt đầu bằng chữ thường
   - Sử dụng camelCase
   - Thường bắt đầu bằng động từ
   Ví dụ:
   ```java
   public void calculateTotal() { }
   public String getFullName() { }
   public boolean isEligible() { }
   ```

## 1. Cơ bản về Java

### 1.1. Cấu trúc chương trình Java cơ bản (Hello World)
```java
public class HelloWorld {
    public static void main(String[] args) {
        System.out.println("Hello, World!");
    }
}
```

Giải thích:
```java
// 1. Khai báo package (không bắt buộc)
package com.example;

// 2. Import các thư viện cần thiết (không bắt buộc)
import java.util.Scanner;

// 3. Khai báo class (BẮT BUỘC)
public class HelloWorld {
    // 4. Phương thức main (BẮT BUỘC để chạy chương trình)
    public static void main(String[] args) {
        // 5. Code chính của chương trình
        System.out.println("Hello, World!");
    }
}
```

Giải thích chi tiết:
1. **package com.example;**
   - Package là cách Java tổ chức code thành các thư mục
   - Giống như đường dẫn thư mục trên máy tính
   - Giúp tránh xung đột tên class giữa các dự án khác nhau
   - Quy ước đặt tên: tất cả viết thường, các từ cách nhau bằng dấu chấm
   - Ví dụ: `com.company.project` sẽ tương ứng với thư mục `com/company/project`
   - Không bắt buộc với chương trình nhỏ, nhưng rất quan trọng trong dự án lớn

2. **import java.util.Scanner;**
   - Import cho phép sử dụng các class từ package khác
   - `java.util.Scanner`: đường dẫn đầy đủ đến class Scanner
   - Có thể import nhiều class cùng lúc: `import java.util.*;`
   - Một số package tự động được import (không cần khai báo):
     + `java.lang.*` (chứa String, System,...)
     + Các class trong cùng package
   - Không bắt buộc nếu:
     + Sử dụng đường dẫn đầy đủ (vd: `java.util.Scanner input = new java.util.Scanner(System.in);`)
     + Chỉ dùng các class từ `java.lang` hoặc cùng package

3. **public class HelloWorld**
   - `public`: Cho phép truy cập từ bất kỳ đâu
   - `class`: Khai báo một lớp trong Java
   - `HelloWorld`: Tên của lớp (PHẢI trùng với tên file .java)
   - Mỗi file .java chỉ được có một public class

4. **public static void main(String[] args)**
   - `public`: Cho phép truy cập từ bất kỳ đâu
   - `static`: Phương thức có thể gọi mà không cần tạo đối tượng
   - `void`: Không trả về giá trị
   - `main`: Tên phương thức (PHẢI là main để làm điểm khởi đầu)
   - `String[] args`: Tham số dòng lệnh (có thể đặt tên khác args)

5. **System.out.println("Hello, World!");**
   - `System`: Lớp chứa các phương thức vào/ra chuẩn
   - `out`: Đối tượng xuất dữ liệu ra màn hình
   - `println()`: In ra và xuống dòng (print() sẽ không xuống dòng)
   - Dấu chấm phẩy (;) là BẮT BUỘC ở cuối mỗi câu lệnh

# Những điều BẮT BUỘC trong một chương trình Java:
1. File phải có đuôi .java
2. Tên file phải trùng với tên public class
3. Phải có phương thức main() để chạy chương trình
4. Mỗi câu lệnh phải kết thúc bằng dấu chấm phẩy
5. Các khối code phải nằm trong cặp dấu { }

Lưu ý cho người mới:
- Java phân biệt chữ hoa/thường, vì vậy System ≠ system
- Các dấu ngoặc phải đủ và đúng cặp: (), {}, []
- Nên thụt lề code để dễ đọc (NetBeans sẽ tự động làm điều này)
- Có thể dùng phím Tab để thụt lề
- Ctrl + Space trong NetBeans sẽ gợi ý code

### 1.2. Kiểu dữ liệu
- Kiểu nguyên thủy: `byte`, `short`, `int`, `long`, `float`, `double`, `boolean`, `char`
- Kiểu tham chiếu: `String`, arrays, objects

### 1.3. Biến và hằng số

#### Biến (Variables)
Biến là một vùng nhớ được đặt tên để lưu trữ dữ liệu.

Cú pháp khai báo biến:
```java
kiểu_dữ_liệu tên_biến [= giá_trị_khởi_tạo];
```

Ví dụ:
```java
int age = 25;  // Biến số nguyên
double salary = 5000.50;  // Biến số thực
String name = "John";  // Biến chuỗi
boolean isStudent = true;  // Biến boolean
char grade = 'A';  // Biến ký tự
```

Lưu ý:
- Tên biến phải bắt đầu bằng chữ cái, dấu gạch dưới (_) hoặc dấu đô la ($).
- Tên biến không được trùng với từ khóa của Java.
- Tên biến nên tuân theo quy tắc camelCase (ví dụ: firstName, lastLoginDate).

#### Hằng số (Constants)
Hằng số là biến có giá trị không thay đổi trong suốt chương trình.

Cú pháp khai báo hằng số:
```java
final kiểu_dữ_liệu TÊN_HẰNG_SỐ = giá_trị;
```

Ví dụ:
```java
final double PI = 3.14159265359;
final int MAX_USERS = 100;
final String DATABASE_URL = "jdbc:mysql://localhost:3306/mydb";
```

Lưu ý:
- Từ khóa `final` được sử dụng để khai báo hằng số.
- Tên hằng số thường viết hoa và sử dụng dấu gạch dưới để ngăn cách các từ.
- Giá trị của hằng số phải được khởi tạo khi khai báo và không thể thay đổi sau đó.

#### Phạm vi của biến (Variable Scope)

1. Biến cục bộ (Local Variables):
   - Được khai báo trong một phương thức hoặc khối lệnh.
   - Chỉ có thể truy cập trong phạm vi khai báo.

2. Biến thành viên (Instance Variables):
   - Được khai báo trong một lớp, ngoài các phương thức.
   - Mỗi đối tượng của lớp có một bản sao riêng.

3. Biến tĩnh (Static Variables):
   - Được khai báo với từ khóa `static`.
   - Chỉ có một bản sao duy nhất cho tất cả các đối tượng của lớp.

Ví dụ:
```java
public class VariableExample {
    static int staticVar = 0;  // Biến tĩnh
    int instanceVar;  // Biến thành viên

    public void method() {
        int localVar = 10;  // Biến cục bộ
        System.out.println(localVar);
    }
}
```
#### Ép kiểu (Casting)
Chuyển đổi giá trị từ một kiểu dữ liệu sang kiểu khác. Có hai loại ép kiểu chính:

1. Ép kiểu ngầm định (Widening Casting):
   - Chuyển đổi từ kiểu dữ liệu nhỏ hơn sang kiểu dữ liệu lớn hơn.
   - Được thực hiện tự động bởi Java, không cần khai báo rõ ràng.
   - Không làm mất dữ liệu.
   ```java
   int myInt = 9;
   double myDouble = myInt;  // Tự động ép kiểu từ int sang double
   ```
   Giải thích: Ở đây, giá trị int 9 được tự động chuyển đổi thành 9.0 (double) mà không cần khai báo rõ ràng.

2. Ép kiểu tường minh (Narrowing Casting):
   - Chuyển đổi từ kiểu dữ liệu lớn hơn sang kiểu dữ liệu nhỏ hơn.
   - Phải được khai báo rõ ràng bằng cách đặt kiểu dữ liệu trong dấu ngoặc đơn trước giá trị.
   - Có thể làm mất dữ liệu hoặc độ chính xác.
   ```java
   double myDouble = 9.78;
   int myInt = (int) myDouble;  // Ép kiểu tường minh từ double sang int
   ```
   Giải thích: Ở đây, giá trị double 9.78 bị cắt bỏ phần thập phân và chỉ giữ lại phần nguyên 9 khi chuyển sang int.

Lưu ý: 
- Ép kiểu ngầm định an toàn và không làm mất dữ liệu.
- Ép kiểu tường minh có thể làm mất dữ liệu hoặc độ chính xác, cần cẩn thận khi sử dụng.
- Khi ép kiểu tường minh, lập trình viên phải chịu trách nhiệm đảm bảo tính đúng đắn của dữ liệu sau khi chuyển đổi.

## 2. Hàm (Methods) trong Java

Hàm trong Java, còn được gọi là phương thức (method), là một khối mã thực hiện một nhiệm vụ cụ thể và có thể được gọi từ các phần khác của chương trình.

### 2.1. Cấu trúc cơ bản của hàm

```java
[modifier] returnType methodName([parameterList]) {
    // Method body
    [return statement;]
}
```

- `modifier`: Xác định phạm vi truy cập (public, private, protected) và các đặc điểm khác (static, final, abstract).
- `returnType`: Kiểu dữ liệu của giá trị trả về hoặc `void` nếu không trả về giá trị.
- `methodName`: Tên của hàm, tuân theo quy tắc đặt tên trong Java.
- `parameterList`: Danh sách các tham số (có thể không có).

### 2.2. Các loại hàm

#### 2.2.1. Hàm có giá trị trả về

Hàm này trả về một giá trị cụ thể sau khi thực hiện xong.

```java
public class Calculator {
    public static int sum(int a, int b) {
        return a + b;
    }

    public static void main(String[] args) {
        int result = sum(2, 5);
        System.out.println("Sum: " + result);  // Output: Sum: 7
    }
}
```

#### 2.2.2. Hàm không có giá trị trả về (void)

Hàm này thực hiện một tác vụ nhưng không trả về giá trị.

```java
public class Printer {
    public static void printSum(int a, int b) {
        System.out.println("Sum: " + (a + b));
    }

    public static void main(String[] args) {
        printSum(2, 5);  // Output: Sum: 7
    }
}
```

### 2.3. Tham số và đối số

- Tham số (Parameters): Biến được khai báo trong định nghĩa hàm.
- Đối số (Arguments): Giá trị thực tế được truyền vào khi gọi hàm.

```java
public static void greet(String name) {  // name là tham số
    System.out.println("Hello, " + name);
}

public static void main(String[] args) {
    greet("Alice");  // "Alice" là đối số
}
```

### 2.4. Overloading (Nạp chồng) hàm

Java cho phép định nghĩa nhiều hàm cùng tên nhưng khác nhau về số lượng hoặc kiểu tham số.

```java
public class Calculator {
    public static int add(int a, int b) {
        return a + b;
    }

    public static double add(double a, double b) {
        return a + b;
    }

    public static void main(String[] args) {
        System.out.println(add(5, 3));       // Calls int version
        System.out.println(add(5.5, 3.2));   // Calls double version
    }
}
```

### 2.5. Hàm đệ quy (Recursive method)

Hàm gọi chính nó trong thân hàm.

```java
public class Factorial {
    public static int factorial(int n) {
        if (n == 0 || n == 1) {
            return 1;
        }
        return n * factorial(n - 1);
    }

    public static void main(String[] args) {
        System.out.println(factorial(5));  // Output: 120
    }
}
```

3. Nếu hàm không trả về giá trị, sử dụng `void` làm kiểu trả về.

### 3.3. Vòng lặp for

Vòng lặp for được sử dụng khi bạn biết trước số lần lặp.

Cấu trúc cơ bản:
```java
for (khởi_tạo; điều_kiện; bước_nhảy) {

}
```

Ví dụ 1: In ra các số từ 1 đến 5
```java
for (int i = 1; i <= 5; i++) {
    System.out.print(i + " ");
}
```
Output:
```
1 2 3 4 5 
```

Giải thích:
1. `int i = 1`: Khởi tạo biến đếm i với giá trị 1.
2. `i <= 5`: Điều kiện để tiếp tục vòng lặp. Vòng lặp sẽ chạy khi i còn nhỏ hơn hoặc bằng 5.
3. `i++`: Tăng giá trị của i sau mỗi lần lặp.
4. Trong mỗi lần lặp, giá trị của i được in ra, theo sau bởi một khoảng trắng.
5. Vòng lặp thực hiện 5 lần với i lần lượt là 1, 2, 3, 4, 5.

Ví dụ 2: Tính tổng các số từ 1 đến n
```java
int n = 5;
int sum = 0;
for (int i = 1; i <= n; i++) {
    sum += i;
    System.out.print(i + (i < n ? " + " : " = "));
}
System.out.print(sum);
```
Output:
```
1 + 2 + 3 + 4 + 5 = 15
```

Giải thích:
1. `int n = 5`: Đặt giới hạn trên cho vòng lặp.
2. `int sum = 0`: Khởi tạo biến tổng.
3. Vòng lặp chạy từ 1 đến 5:
   - Mỗi lần lặp, cộng giá trị i vào sum.
   - In ra giá trị i, theo sau bởi "+" nếu chưa phải số cuối cùng, hoặc "=" nếu là số cuối.
4. Sau vòng lặp, in ra tổng cuối cùng.

```
Apple Banana Cherry
```

1. Tạo một mảng `fruits` chứa 3 chuỗi.
2. Vòng lặp for-each sẽ duyệt qua từng phần tử trong mảng `fruits`.
3. Trong mỗi lần lặp:
   - `fruit` sẽ lần lượt nhận giá trị của từng phần tử trong mảng.
   - Giá trị của `fruit` được in ra, theo sau bởi một khoảng trắng.
4. Vòng lặp tự động kết thúc sau khi đã duyệt qua tất cả phần tử trong mảng.

Lưu ý:
- For-each loop rất hữu ích khi bạn muốn duyệt qua tất cả phần tử của một mảng hoặc collection mà không cần quan tâm đến chỉ số.

Ví dụ: Nhập số từ bàn phím cho đến khi nhập số dương

```java
}
```

````java
import java.util.ArrayList;
````

````java
    private String name;
    private int id;

    public Student(String name, int id) {
        this.name = name;
        this.id = id;
    }

    public int getId() { return id; }

    public String toString() {
        return "Student{name='" + name + "', id=" + id + "}";
    }
}

public class StudentManager {
    public static void main(String[] args) {
        ArrayList<Student> students = new ArrayList<>();

        // Thêm sinh viên
        students.add(new Student("Alice", 101));
        students.add(new Student("Bob", 102));
        students.add(new Student("Charlie", 103));

        // In danh sách sinh viên
        System.out.println("Danh sách sinh viên:");
        for (Student student : students) {
            System.out.println(student);
        }

        // Sắp xếp theo tên
        Collections.sort(students, Comparator.comparing(Student::getName));
        System.out.println("\nDanh sách sau khi sắp xếp theo tên:");
        for (Student student : students) {
            System.out.println(student);
        }

        // Tìm kiếm sinh viên theo ID
        int searchId = 102;
        for (Student student : students) {
            if (student.getId() == searchId) {
                System.out.println("\nTìm thấy sinh viên có ID " + searchId + ": " + student);
                break;
            }
        }
    }
}
``````

### 5.7. Bài tập thực hành

<details>
<summary>Click vào để xem đáp án</summary>

```java
public static boolean isPalindrome(String str) {
    str = str.toLowerCase().replaceAll("[^a-zA-Z0-9]", "");
    int left = 0;
    int right = str.length() - 1;
    while (left < right) {
        if (str.charAt(left) != str.charAt(right)) {
            return false;
        }
        left++;
        right--;
    }
    return true;
}
```

## 6. OOP (Object-Oriented Programming) trong Java

OOP là một phương pháp lập trình dựa trên khái niệm về "objects" (đối tượng), có thể chứa dữ liệu và mã.

### 6.1. Class và Object

Class là một bản thiết kế hoặc prototype từ đó các objects được tạo ra.

`````java
public class Car {
    // Thuộc tính (fields)
    private String brand;
    private int year;

    // Constructor
    public Car(String brand, int year) {
        this.brand = brand;
        this.year = year;
    }

    // Phương thức (method)
    public void start() {
        System.out.println(brand + " is starting...");
    }

    // Getters và Setters
    public String getBrand() {
        return brand;
    }

    public void setBrand(String brand) {
        this.brand = brand;
    }

    // ... (getters và setters cho year)
}

// Tạo object
Car myCar = new Car("Toyota", 2020);
myCar.start();
`````


Giải thích:
- `class Car`: Định nghĩa một lớp Car.
- `private String brand`: Thuộc tính private, chỉ có thể truy cập trong class.
- `public Car(String brand, int year)`: Constructor để khởi tạo object.
- `public void start()`: Phương thức public, có thể gọi từ bên ngoài class.
- `Car myCar = new Car("Toyota", 2020)`: Tạo một object Car mới.

### 6.2. Kế thừa (Inheritance)

Kế thừa cho phép một class kế thừa thuộc tính và phương thức từ một class khác.

`````java
public class ElectricCar extends Car {
    private int batteryCapacity;

    public ElectricCar(String brand, int year, int batteryCapacity) {
        super(brand, year);  // Gọi constructor của lớp cha
        this.batteryCapacity = batteryCapacity;
    }

    @Override
    public void start() {
        System.out.println(getBrand() + " is starting silently...");
    }

    public void charge() {
        System.out.println("Charging the electric car...");
    }
}

ElectricCar myTesla = new ElectricCar("Tesla", 2023, 100);
myTesla.start();  // Sử dụng phương thức đã được override
myTesla.charge();  // Sử dụng phương thức mới
`````


Giải thích:
- `extends Car`: ElectricCar kế thừa từ Car.
- `super(brand, year)`: Gọi constructor của lớp cha.
- `@Override`: Chú thích cho biết phương thức này ghi đè phương thức của lớp cha.

### 6.3. Interface

Interface định nghĩa một bộ các phương thức abstract mà các class implement nó phải triển khai.

`````java
public interface Chargeable {
    void charge();
    default void displayChargeStatus() {
        System.out.println("Displaying charge status...");
    }
}

public class ElectricCar extends Car implements Chargeable {
    // ... (các thuộc tính và phương thức khác)

    @Override
    public void charge() {
        System.out.println("Charging the electric car...");
    }
}

Chargeable myChargeable = new ElectricCar("Tesla", 2023, 100);
myChargeable.charge();
myChargeable.displayChargeStatus();
`````


Giải thích:
- `interface Chargeable`: Định nghĩa một interface.
- `void charge();`: Phương thức abstract trong interface.
- `default void displayChargeStatus()`: Phương thức mặc định trong interface (từ Java 8).
- `implements Chargeable`: ElectricCar triển khai interface Chargeable.

### 6.4. Tính đa hình (Polymorphism)

Đa hình cho phép sử dụng các class con thông qua tham chiếu của class cha.

### 6.5. Encapsulation (Đóng gói)

Encapsulation là việc ẩn các chi tiết triển khai bên trong và chỉ hiển thị chức năng cho người dùng.

`````java
public class Car {
    private int speed;  // private field

    public void setSpeed(int speed) {
        if (speed >= 0 && speed <= 200) {
            this.speed = speed;
        } else {
            System.out.println("Invalid speed");
        }
    }

    public int getSpeed() {

}
`````


Giải thích:
- `private int speed`: Thuộc tính private, không thể truy cập trực tiếp từ bên ngoài.
- `setSpeed()` và `getSpeed()`: Phương thức public để truy cập và thay đổi thuộc tính private.

### 6.6. Abstraction

Abstraction là quá trình ẩn các chi tiết triển khai phức tạp và chỉ hiển thị chức năng cần thiết.

`````java
public abstract class Vehicle {
    public abstract void move();

    public void displayInfo() {
        System.out.println("This is a vehicle.");
    }
}

public class Car extends Vehicle {
    @Override
    public void move() {
        System.out.println("Car is moving on the road.");
    }
}
`````


Giải thích:
- `abstract class Vehicle`: Lớp trừu tượng, không thể tạo đối tượng trực tiếp.
- `public abstract void move();`: Phương thức trừu tượng, phải được triển khai bởi lớp con.

OOP trong Java cung cấp một cách mạnh mẽ để tổ chức và cấu trúc code, làm cho nó dễ bảo trì, mở rộng và tái sử dụng.

## 7. Exception Handling

Exception Handling là cơ chế xử lý các tình huống bất thường (ngoại lệ) có thể xảy ra trong quá trình thực thi chương trình.

### 7.1. Cấu trúc try-catch-finally

```java
try {
    // Khối code có thể gây ra ngoại lệ
    int result = 10 / 0;  // Ví dụ: chia cho 0 sẽ gây ra ArithmeticException
} catch (ArithmeticException e) {
    // Xử lý ngoại lệ cụ thể
    System.out.println("Lỗi: Không thể chia cho 0!");
    e.printStackTrace();  // In ra thông tin chi tiết về ngoại lệ
} catch (Exception e) {
    // Xử lý các ngoại lệ khác
    System.out.println("Đã xảy ra lỗi: " + e.getMessage());
} finally {
    // Khối code luôn được thực thi, bất kể có ngoại lệ hay không
    System.out.println("Khối finally luôn được thực thi");
}
```

Giải thích:
- `try`: Chứa code có thể gây ra ngoại lệ.
- `catch`: Bắt và xử lý các ngoại lệ cụ thể. Có thể có nhiều khối catch cho các loại ngoại lệ khác nhau.
- `finally`: Chứa code luôn được thực thi, dù có ngoại lệ hay không. Thường dùng để giải phóng tài nguyên.

### 7.2. Throws và Throw

```java
public void readFile(String filename) throws IOException {

    // Code đọc file
}
```

- `throws`: Khai báo rằng một phương thức có thể ném ra ngoại lệ.
- `throw`: Ném ra một ngoại lệ cụ thể.

### 7.3. Try-with-resources

```java
try (FileReader fr = new FileReader("file.txt");
```

## 9. Collections Framework

Collections Framework trong Java cung cấp một kiến trúc để lưu trữ và thao tác với nhóm các đối tượng. Framework này bao gồm các interface, các lớp triển khai và các thuật toán.

List là một interface trong Collections Framework đại diện cho một danh sách có thứ tự các phần tử. Các phần tử trong List có thể truy cập bằng chỉ số và có thể chứa các phần tử trùng lặp.

```java
import java.util.*;

List<String> list = new ArrayList<>();

// Kiểm tra kích thước của List
```

Map là một interface đại diện cho một cấu trúc dữ liệu lưu trữ các cặp key-value. Mỗi key trong Map là duy nhất.

### 9.4. Collections Utility Class

Lớp `Collections` cung cấp các phương thức static để thao tác với các collection.

```java

// Tìm giá trị lớn nhất và nhỏ nhất

// Đếm số lần xuất hiện của một phần tử
```

  - Sử dụng Map khi cần lưu trữ và truy xuất dữ liệu theo cặp key-value.
- Các phương thức của lớp Collections thường yêu cầu collection là mutable (có thể thay đổi). Đối với các immutable collection, cần tạo một bản sao mutable trước khi áp dụng các phương thức này.

### Sắp xếp mảng tăng dần

```java
public static void bubbleSort(int[] arr) {
    int n = arr.length;
    for (int i = 0; i < n-1; i++)
        for (int j = 0; j < n-i-1; j++)
```

## 1 Định nghĩa và khai báo lớp
### 1.1 Khai báo lớp
- **public**: Cho phép lớp này được sử dụng bởi các gói khác.
- **abstract**: Đánh dấu lớp này là trừu tượng, không thể tạo đối tượng từ lớp này.
- **final**: Đánh dấu lớp này không thể bị kế thừa.
- **class**: Từ khóa để khai báo một lớp. Ví dụ: `class LopThu` thì lớp này có tên là `LopThu`.
- **extends**: Từ khóa để chỉ ra lớp này kế thừa từ lớp khác.
- **implements**: Từ khóa để chỉ ra lớp này triển khai các giao diện (Interfaces).
- Vui lòng xem bài 3 để biết chi tiết hơn ở dưới.
### 1.2 Khai báo thuộc tính
Một ví dụ khai báo thuộc tính như sau:

```java
public class LopThu {
    private String moiTruongSong;
    private boolean laDongVatHangNhiet;
    // Các thuộc tính khác
}
```

Trong lớp **LopThu** trên, ta có thể tổng quát việc khai báo thuộc tính như sau:

```java
[Access Level] [Static] [Type] [Tên thuộc tính];
```

_Trong đó:_

- **Access Level**: Có thể là `public`, `private`, `protected`, hoặc có thể bỏ trống (mặc định là package-private). Ý nghĩa của các từ khóa này đã được mô tả ở phần trên.
- **Static**: Từ khóa chỉ ra rằng đây là một thuộc tính lớp, dùng chung cho cả lớp, không thuộc về riêng một đối tượng nào.
- **Type**: Kiểu dữ liệu của thuộc tính.
- **Tên thuộc tính**: Tên của thuộc tính.

> **Lưu ý**: 
> - Khai báo thuộc tính là tất cả các khai báo nằm trong thân của một lớp và bên ngoài tất cả các phương thức và hàm tạo. 
> - Khai báo ở những chỗ khác sẽ cho ta biến cục bộ hoặc tham số.
### 1.3 Khai báo phương thức

Một ví dụ khai báo phương thức như sau:

```java
public class LopThu {
    // Phương thức
    public void hienThiThongTin() {
        System.out.println("Thông tin lớp thú");
    }
}
```

Cấu trúc tổng quát của một phương thức:

```java
[Access Level] [Static] [ReturnType] [Tên phương thức]([Danh sách tham số]) {
    // Thân phương thức
}
```

_Trong đó:_

- **Access Level**: Có thể là `public`, `private`, `protected`, hoặc có thể bỏ trống (mặc định là package-private). Quy định phạm vi truy cập của phương thức.
- **Static**: Từ khóa chỉ ra rằng phương thức này thuộc về lớp, không thuộc về riêng một đối tượng nào.
- **ReturnType**: Kiểu dữ liệu của giá trị trả về. Dùng `void` nếu phương thức không trả về giá trị.
- **Tên phương thức**: Tên của phương thức, nên bắt đầu bằng động từ và sử dụng camelCase.
- **Danh sách tham số**: Danh sách các tham số mà phương thức nhận vào, có thể không có tham số.

> **Lưu ý**: 
> - Phương thức có thể có hoặc không có tham số.
> - Phương thức có thể trả về một giá trị hoặc không (dùng `void`).
> - Tên phương thức nên rõ ràng và mô tả được chức năng của nó.

## 3 Cấp độ truy cập trong Java

Java cung cấp hai loại cấp độ truy cập: Access Modifier và Non-access Modifier.

### Access Modifier

Access Modifier xác định phạm vi truy cập của biến, phương thức, constructor hoặc lớp. Có bốn loại Access Modifier:

1. **private**: Chỉ truy cập được trong chính lớp đó.
2. **default**: Truy cập được trong cùng package.
3. **protected**: Truy cập được trong cùng package và lớp con khác package.
4. **public**: Truy cập được từ bất kỳ đâu.

### Non-access Modifier

Non-access Modifier bao gồm các từ khóa như `static`, `abstract`, `synchronized`, `native`, `volatile`, `transient`,...

### Bảng tổng quan về Access Modifier

| Từ khóa    | Trong lớp | Lớp con cùng gói | Lớp con khác gói | Lớp khác cùng gói | Lớp khác khác gói |
|------------|-----------|------------------|------------------|-------------------|-------------------|
| private    | X         | -                | -                | -                 | -                 |
| default    | X         | X                | -                | X                 | -                 |
| protected  | X         | X                | X                | X                 | -                 |
| public     | X         | X                | X                | X                 | X                 |

### Ví dụ về các phạm vi truy cập

#### Phạm vi truy cập `private`

```java
class Alpha {
    private int bienPrivate;
    private void privateMethod() {
        System.out.println("Đây là phương thức private");
    }
}

class Beta {
    void accessMethod() {
        Alpha a = new Alpha();
        // a.bienPrivate = 10; // Không hợp lệ
        // a.privateMethod(); // Không hợp lệ
    }
}
```

> **Lưu ý**: Một lớp không thể là `private` hoặc `protected`, ngoại trừ lớp lồng nhau.

#### Phạm vi truy cập `default`

```java
// Lưu file với tên Alpha.java
package package1;

class Alpha {
    void defaultMethod() {
        System.out.println("Đây là phương thức default");
    }
}

// Lưu file với tên Beta.java
package package2;

class Beta {
    public static void main(String[] args) {
        Alpha obj = new Alpha();
        // obj.defaultMethod(); // Không hợp lệ
    }
}
```

#### Phạm vi truy cập `protected`

```java
// Lưu file với tên Alpha.java
package package1;

class Alpha {
    protected void protectedMethod() {
        System.out.println("Đây là phương thức protected");
    }
}

// Lưu file với tên Beta.java
package package2;

class Beta extends Alpha {
    public static void main(String[] args) {

    }
```

#### Phạm vi truy cập `public`

```java
public class Alpha {
    public int bienPublic;

    }
}

class Beta {
    void accessMethod() {
        Alpha a = new Alpha();
        a.bienPublic = 10; // Hợp lệ
        a.publicMethod(); // Hợp lệ
    }
}
```

## 4 Tại sao lại nên xài Constructor?

### Đặc điểm của Constructor

- **Tên trùng với tên lớp**: Constructor có tên giống với tên lớp và không có kiểu trả về.
- **Khởi tạo đối tượng**: Constructor được sử dụng để khởi tạo đối tượng, thiết lập giá trị ban đầu cho các thuộc tính.
- **Tự động gọi**: Constructor được gọi tự động khi một đối tượng được tạo ra.
- **Có thể có tham số**: Cho phép truyền tham số để khởi tạo đối tượng với các giá trị khác nhau.
- **Overloading**: Có thể có nhiều constructor trong một lớp với danh sách tham số khác nhau.

### Constructor Mặc Định

- **Constructor mặc định**: Được Java cung cấp tự động nếu không có constructor nào được khai báo. Nó không có tham số và không thực hiện hành động gì đặc biệt.
- **Ví dụ**:
  ```java
  class Student {
      Student() { // Constructor mặc định
          System.out.println("Đang tạo student");
      }
  }
  ```

### Constructor Overloading

- **Constructor Overloading**: Cho phép tạo nhiều constructor trong cùng một lớp với danh sách tham số khác nhau.
- **Ví dụ**:
  ```java
  class Student {
      int id;
      String name;

      Student(int i, String n) {
          id = i;
          name = n;
      }

      Student(int i, String n, int a) {
          id = i;
          name = n;
          age = a;
      }
  }
  ```

### Sự Khác Nhau Giữa Constructor và Phương Thức

| Constructor                                                                        | Phương thức                                                       |
| ---------------------------------------------------------------------------------- | ----------------------------------------------------------------- |
| Được sử dụng để khởi tạo trạng thái của một đối tượng.                             | Được sử dụng để thể hiện hành động của một đối tượng.             |

| Java tạo ra constructor mặc định nếu không có constructor nào.                     | Không được tạo ra bởi trình biên dịch Java.                       |

Constructor là một phần quan trọng trong Java, giúp đảm bảo rằng đối tượng được khởi tạo đúng cách và có thể được tùy chỉnh thông qua overloading.

## 5 String nâng cấp:
### 5.1 Lớp StringBuffer

Lớp StringBuffer trong Java là một công cụ mạnh mẽ để làm việc với chuỗi, cho phép thay đổi và nối chuỗi một cách hiệu quả. Dưới đây là lý do và thời điểm bạn nên sử dụng StringBuffer:
#### Khi nào nên sử dụng StringBuffer?
1. Khi cần thay đổi chuỗi thường xuyên: Nếu bạn cần thực hiện nhiều thao tác thay đổi trên chuỗi, như nối, chèn, xóa, hoặc thay thế ký tự, StringBuffer là lựa chọn tốt hơn so với String, vì String là bất biến (immutable).
2. Khi làm việc trong môi trường đa luồng: StringBuffer là đồng bộ (synchronized), nghĩa là nó an toàn khi sử dụng trong các ứng dụng đa luồng. Nếu bạn cần thao tác với chuỗi trong một ứng dụng đa luồng, StringBuffer sẽ đảm bảo rằng các thao tác trên chuỗi là an toàn.
#### Vì sao nên sử dụng StringBuffer?
1. Hiệu suất tốt hơn với thao tác thay đổi chuỗi: StringBuffer cho phép thay đổi nội dung của chuỗi mà không cần tạo ra đối tượng mới mỗi khi thay đổi, giúp tiết kiệm bộ nhớ và cải thiện hiệu suất.
2. Đồng bộ hóa: StringBuffer cung cấp các phương thức đồng bộ, đảm bảo rằng các thao tác trên chuỗi là an toàn trong môi trường đa luồng, tránh các vấn đề về đồng bộ hóa dữ liệu.
3. Khả năng mở rộng: StringBuffer có thể tự động mở rộng dung lượng khi cần thiết, giúp bạn dễ dàng quản lý và thao tác với các chuỗi lớn mà không cần lo lắng về việc cấp phát bộ nhớ thủ công.

### Khởi tạo một đối tượng StringBuffer

Lớp `StringBuffer` cung cấp nhiều cách khởi tạo đối tượng, dưới đây là các ví dụ:

- **Sử dụng constructor mặc định:**
  ```java
  StringBuffer s1 = new StringBuffer();
  ```

- **Sử dụng constructor với dung lượng ban đầu:**
  ```java
  StringBuffer s2 = new StringBuffer(20);
  ```

- **Sử dụng constructor với một chuỗi ban đầu:**
  ```java
  StringBuffer s3 = new StringBuffer("StringBuffer");
  ```

Trong ví dụ trên, `s3` sẽ chứa chuỗi "StringBuffer".

```java
System.out.println("s3 = " + s3); // s3 = StringBuffer
System.out.println(s2.length());  // 0
System.out.println(s3.length());  // 12
System.out.println(s1.capacity());  // 16
System.out.println(s2.capacity());  // 20
System.out.println(s3.capacity());  // 28
```

Hãy lưu ý rằng `length()` và `capacity()` của `StringBuffer` là hai phương thức khác nhau. `length()` trả về số ký tự thực sự trong `StringBuffer`, trong khi `capacity()` trả về dung lượng tối đa mà `StringBuffer` có thể chứa mà không cần tăng cấp thêm.

Dung lượng của `StringBuffer` có thể thay đổi bằng cách sử dụng phương thức `ensureCapacity()`. Nếu dung lượng mới được yêu cầu lớn hơn dung lượng hiện tại, nó sẽ được tăng lên theo cơ chế:

```
NewCapacity = OldCapacity * 2 + 2
```

Ví dụ về `ensureCapacity()`:

```java
StringBuffer s1 = new StringBuffer(5);
System.out.println("Initial capacity of s1 = " + s1.capacity()); // 5
s1.ensureCapacity(8);
System.out.println("Capacity of s1 after ensureCapacity(8) = " + s1.capacity()); // 12
s1.ensureCapacity(30);
System.out.println("Capacity of s1 after ensureCapacity(30) = " + s1.capacity()); // 30
```

### Các phương thức của lớp StringBuffer

- **append()**: Dùng để nối thêm chuỗi, mảng ký tự hoặc bất kỳ kiểu dữ liệu nào vào cuối của `StringBuffer`.
  ```java
  StringBuffer s1 = new StringBuffer("Good");
  s1.append(" evening");
  // Kết quả: "Good evening"
  ```

- **insert()**: Cho phép chèn một chuỗi, một ký tự, một giá trị nguyên hoặc số thực vào vị trí bất kỳ trong `StringBuffer`.
  ```java
  StringBuffer str = new StringBuffer("Java sion");
  str.insert(1, 'b');
  // Kết quả: "Jbava sion"
  ```

- **charAt()**: Trả về ký tự tại một vị trí xác định trong `StringBuffer`.
  ```java
  StringBuffer str = new StringBuffer("James Gosling");
  char letter = str.charAt(6); // letter chứa 'G'
  ```

- **setCharAt()**: Dùng để thay thế ký tự tại một vị trí cụ thể trong `StringBuffer` bằng một ký tự khác.
  ```java
  StringBuffer name = new StringBuffer("Jawa");
  name.setCharAt(2, 'v');
  // Kết quả: "Java"
  ```

- **setLength()**: Dùng để thiết lập chiều dài của `StringBuffer`. Nếu chiều dài mới nhỏ hơn chiều dài hiện tại, các ký tự thừa sẽ bị cắt bỏ. Nếu chiều dài mới lớn hơn, các ký tự null sẽ được thêm vào cuối.
  ```java
  StringBuffer str = new StringBuffer(10);
  str.setLength(str.length() + 10);
  ```

- **getChars()**: Dùng để trích xuất các ký tự từ `StringBuffer` và sao chép chúng vào một mảng.
  ```java
  StringBuffer str = new StringBuffer("Leopard");
  char[] ch = new char[10];
  str.getChars(3, 6, ch, 0);
  // Kết quả: mảng ch chứa "par"
  ```

- **reverse()**: Dùng để đảo ngược nội dung của `StringBuffer` và trả về một đối tượng `StringBuffer` mới.
  ```java
  StringBuffer str = new StringBuffer("devil");
  StringBuffer strrev = str.reverse();
  // Kết quả: "lived"
  ```

### 5.2 String.split()

Phương thức `String.split()` trong Java được sử dụng để phân tách chuỗi thành một mảng các chuỗi con dựa trên một biểu thức chính quy (regex). Đây là một công cụ mạnh mẽ và linh hoạt cho việc xử lý chuỗi.

#### Cách sử dụng `String.split()`

1. **Phân tách bằng một ký tự đơn giản:**

   ```java
   String str = "apple,banana,orange";
   String[] fruits = str.split(",");
   // Kết quả: ["apple", "banana", "orange"]
   ```

2. **Phân tách bằng nhiều ký tự hoặc biểu thức chính quy:**

   ```java
   String str = "one:two;three,four";
   String[] parts = str.split("[:;,]");
   // Kết quả: ["one", "two", "three", "four"]
   ```

3. **Giới hạn số lượng phần tử:**

   Bạn có thể chỉ định số lượng phần tử tối đa mà mảng kết quả có thể chứa. Phần còn lại của chuỗi sẽ được gộp vào phần tử cuối cùng.

   ```java
   String str = "a,b,c,d,e";
   String[] result = str.split(",", 3);
   // Kết quả: ["a", "b", "c,d,e"]
   ```

#### Lưu ý khi sử dụng `String.split()`

- **Biểu thức chính quy**: `split()` sử dụng biểu thức chính quy, vì vậy các ký tự đặc biệt như `.` hoặc `|` cần được thoát (`\\.` hoặc `\\|`).
- **Chuỗi rỗng**: Nếu chuỗi đầu vào là rỗng, `split()` sẽ trả về một mảng chứa một phần tử rỗng.

#### Ví dụ thực tế

```java
public class SplitExample {
    public static void main(String[] args) {
        String sentence = "Java is fun";
        String[] words = sentence.split(" ");
        
        for (String word : words) {
            System.out.println(word);
        }
        // Output:
        // Java
        // is
        // fun
    }
}
```

## 6 Mô hình MVC trong Java

Mô hình MVC (Model-View-Controller) là một kiến trúc phần mềm phổ biến được sử dụng trong lập trình Java. Nó giúp tách biệt các thành phần chính của ứng dụng, giúp mã nguồn dễ bảo trì và mở rộng hơn.

### Các thành phần của MVC

Mô hình MVC bao gồm 3 thành phần chính:

1. **Model**:
   - Chịu trách nhiệm về dữ liệu và logic nghiệp vụ của ứng dụng.
   - Thực hiện các thao tác với cơ sở dữ liệu như lưu trữ, truy xuất và cập nhật dữ liệu.
   - Model không biết gì về View hoặc Controller, nó chỉ quan tâm đến việc quản lý dữ liệu.

2. **View**:
   - Chịu trách nhiệm hiển thị dữ liệu cho người dùng.
   - Nhận dữ liệu từ Model và trình bày nó trên giao diện người dùng.
   - View không chứa bất kỳ logic nghiệp vụ nào, nó chỉ đơn thuần là hiển thị dữ liệu.

3. **Controller**:
   - Đóng vai trò điều khiển luồng dữ liệu giữa Model và View.
   - Nhận đầu vào từ người dùng thông qua View, xử lý nó và cập nhật Model tương ứng.
   - Sau đó, Controller sẽ yêu cầu View cập nhật giao diện dựa trên dữ liệu mới từ Model.

### Tổ chức thư mục

Một dự án Java sử dụng mô hình MVC thường được tổ chức như sau:

```
project-root/
│
├── src/
│   ├── model/
│   │   └── User.java
│   │   └── Product.java
│   │   └── Database.java
│   │
│   ├── view/
│   │   └── UserView.java
│   │   └── ProductView.java
│   │
│   ├── controller/
│   │   └── UserController.java
│   │   └── ProductController.java
│   │
│   └── Main.java
│
└── resources/
    └── config.properties
```

Như bạn thấy, các thành phần Model, View và Controller được tổ chức thành các gói (packages) riêng biệt, giúp dễ dàng quản lý và bảo trì mã nguồn.

### Lợi ích của MVC

Mô hình MVC mang lại nhiều lợi ích cho việc phát triển ứng dụng Java, bao gồm:

1. **Tách biệt rõ ràng**: Mô hình MVC giúp tách biệt logic nghiệp vụ, giao diện và điều khiển, làm cho mã nguồn dễ bảo trì và mở rộng hơn.

2. **Tái sử dụng**: Các thành phần trong mô hình MVC (Model, View, Controller) có thể được tái sử dụng trong các ứng dụng khác.

3. **Dễ dàng kiểm thử**: Với sự tách biệt rõ ràng, từng thành phần trong mô hình MVC có thể được kiểm thử độc lập.

## 8. Cách Sử Dụng Exception trong Java

Trong Java, Exception là một cơ chế quan trọng để xử lý các tình huống bất thường có thể xảy ra trong quá trình chạy chương trình. Việc sử dụng Exception giúp duy trì luồng chạy bình thường của ứng dụng, thay vì kết thúc chương trình một cách đột ngột.

### 8.1. Cấu Trúc Try-Catch-Finally

Cơ chế xử lý Exception chính trong Java là sử dụng khối `try-catch-finally`:

```java
try {
    // Các câu lệnh có thể gây ra Exception
    int result = 10 / 0; // Ví dụ: chia cho 0 sẽ gây ra ArithmeticException
} catch (ArithmeticException e) {
    // Xử lý Exception cụ thể
    System.out.println("Lỗi: Không thể chia cho 0!");
    e.printStackTrace();
} finally {
    // Khối lệnh luôn được thực thi, bất kể có Exception hay không
    System.out.println("Khối finally luôn được thực thi");
}
```

Giải thích:
- `try`: Chứa các câu lệnh có thể gây ra Exception.
- `catch`: Bắt và xử lý các Exception cụ thể. Có thể có nhiều khối `catch` để xử lý các loại Exception khác nhau.
- `finally`: Chứa các câu lệnh luôn được thực thi, dù có Exception hay không. Thường dùng để giải phóng tài nguyên.

### 8.2. Ném và Khai Báo Exception

Để ném một Exception, ta sử dụng từ khóa `throw`:

```java
public void processData(String data) {
    if (data == null || data.isEmpty()) {
        throw new IllegalArgumentException("Dữ liệu không hợp lệ");
    }
    // Xử lý dữ liệu
}
```

Để khai báo rằng một phương thức có thể ném ra một Exception, ta sử dụng từ khóa `throws`:

```java
public void readFile(String filename) throws IOException {
    File file = new File(filename);
    FileReader reader = new FileReader(file);
    // Đọc dữ liệu từ file
}
```

Khi gọi phương thức `readFile()`, ta cần xử lý Exception `IOException` bằng cách sử dụng `try-catch` hoặc khai báo `throws` trong phương thức gọi.

### 8.3. Ví Dụ: Xử Lý NumberFormatException

Giả sử ta có một phương thức để chuyển đổi một chuỗi thành số nguyên:

```java
public static int convertToInt(String str) {
    return Integer.parseInt(str);
}
```

Nếu người dùng nhập vào một chuỗi không phải số, phương thức `parseInt()` sẽ ném ra `NumberFormatException`:

```java
public static void main(String[] args) {
    String input = "abc";
    try {
        int value = convertToInt(input);
        System.out.println("Giá trị đã chuyển đổi: " + value);
    } catch (NumberFormatException e) {
        System.out.println("Lỗi: Chuỗi không phải số");
    }
}
```

Kết quả:
```
Lỗi: Chuỗi không phải số
```

Trong ví dụ này, chúng ta đã bắt và xử lý `NumberFormatException` để hiển thị một thông báo lỗi thân thiện với người dùng.

### 8.4. Ví Dụ: Xử Lý FileNotFoundException

Giả sử ta có một phương thức để đọc dữ liệu từ một tệp tin:

```java
public static void readDataFromFile(String filename) throws FileNotFoundException, IOException {
    File file = new File(filename);
    FileReader reader = new FileReader(file);
    BufferedReader bufferedReader = new BufferedReader(reader);
    
    String line;
    while ((line = bufferedReader.readLine()) != null) {
        System.out.println(line);
    }
    
    reader.close();
}
```

Nếu tệp tin không tồn tại, phương thức `FileReader` sẽ ném ra `FileNotFoundException`. Nếu có lỗi xảy ra trong quá trình đọc, `readLine()` sẽ ném ra `IOException`.

Trong phương thức `main()`, chúng ta cần xử lý các Exception này:

```java
public static void main(String[] args) {
    try {
        readDataFromFile("data.txt");
    } catch (FileNotFoundException e) {
        System.out.println("Tệp tin không tồn tại: " + e.getMessage());
    } catch (IOException e) {
        System.out.println("Lỗi khi đọc tệp tin: " + e.getMessage());
    }
}
```

Kết quả:
```
Tệp tin không tồn tại: data.txt (The system cannot find the file specified)
```

Trong ví dụ này, chúng ta đã xử lý cả `FileNotFoundException` và `IOException` để cung cấp thông báo lỗi cụ thể cho người dùng.
