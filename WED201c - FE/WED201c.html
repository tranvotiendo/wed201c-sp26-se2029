Phân biệt ID và Class

Trong HTML, thuộc tính "id" được dùng để xác định một định danh duy nhất cho mỗi phần tử trên trang web. Mỗi phần tử chỉ được phép có một "id" và giá trị này phải là duy nhất trong toàn bộ tài liệu HTML. Thuộc tính id thường được sử dụng trong CSS hoặc JavaScript để thao tác hoặc định dạng riêng cho phần tử đó một cách chính xác.

Ngược lại, thuộc tính "class" dùng để nhóm nhiều phần tử HTML lại với nhau, giúp chúng có thể chung một kiểu dáng hoặc hành vi. Nhiều phần tử có thể được gán chung một hoặc nhiều class, cho phép áp dụng định dạng CSS hay thao tác JavaScript trên tất cả những phần tử này cùng lúc. Một phần tử HTML có thể có nhiều class khác nhau.

Điểm khác biệt chính:

  • ID: Dùng để định danh duy nhất; một trang chỉ có thể có một phần tử mang một ID cụ thể.
  • Class: Dùng để nhóm nhiều phần tử có cùng tính chất; nhiều phần tử có thể cùng mang một class và một phần tử có thể có nhiều class.
Ví dụ: 
<div id="header">Đây là phần đầu trang</div>
<p class="text-highlight">Đoạn văn có định dạng đặc biệt</p>
<p class="text-highlight">Một đoạn văn khác cũng dùng class này</p>

3.1 Giới thiệu CSS và cách nhúng

CSS (Cascading Style Sheets) dùng để định dạng giao diện web: màu sắc, font chữ, khoảng cách, bố cục…

1. Inline

Trực tiếp trong thẻ (chỉ dùng cho style nhỏ).

2. Internal

Trong thẻ <style> ở phần <head>.

3. External

File .css riêng (Khuyên dùng).

<link rel="stylesheet"

3.2 Selector trong CSS

Loại Selector Ví dụ Mô tả
Element selector p { ... } Chọn theo tên thẻ HTML.
Class selector .highlight { ... } Chọn theo class (có thể chọn nhiều phần tử).
ID selector #header { ... } Chọn phần tử cụ thể duy nhất.
Attribute selector input[type="text"] Chọn theo thuộc tính.
Pseudo-classes a:hover { ... } Chọn trạng thái phần tử (di chuột, focus...).

3.5 Layout cơ bản và Flexbox

position: static & relative

Static (Default)
Relative (Moved)

Static: Luồng bình thường, bỏ qua top/left.
Relative: Dịch chuyển so với vị trí gốc nhưng vẫn giữ chỗ.

position: absolute

Absolute
Parent (Relative)

Absolute: Đặt tuyệt đối theo cha gần nhất có position (không bám luồng).

position: fixed - Luôn dính trên đầu màn hình, kể cả khi bạn cuộn trang (hay dùng làm navbar cố định).

Flexbox linh hoạt

Flex container: Phần tử cha (display: flex). Quản lý việc sắp xếp, căn chỉnh các con.

Flex items: Các con trực tiếp. Có thể co giãn (grow/shrink) theo quy tắc.

Main axis (Trục chính): Hướng sắp xếp (ngang nếu row, dọc nếu column).

Cross axis (Trục chéo): Trục vuông góc với trục chính.

.container {
  display: flex;
  flex-direction: row; /* hoặc column */
  justify-content: space-between; /* Căn trục chính */
  align-items: center; /* Căn trục chéo */
  flex-wrap: wrap; /* Cho phép xuống dòng */
}

4.6 Xử lý form và validation

function validateForm() {

5.1 Nguyên lý responsive, mobile-first

Responsive Web Design là kỹ thuật giúp giao diện tự thích ứng với nhiều kích thước màn hình (mobile, tablet, desktop) mà không cần làm nhiều phiên bản trang khác nhau.

Triết lý “mobile-first”: thiết kế và viết CSS cho màn hình nhỏ trước (mobile), sau đó dùng media queries để mở rộng dần cho tablet, desktop; giúp tối ưu hiệu năng và trải nghiệm trên thiết bị di động.

Các thành phần cốt lõi của responsive:

  • Viewport meta: <meta name="viewport" content="width=device-width, initial-scale=1.0"> để trình duyệt hiểu đúng chiều rộng thiết bị.
  • Layout fluid: Dùng đơn vị %, vw, flex, grid thay vì cố định px cho chiều rộng chính.
  • Breakpoints: Đặt các “ngưỡng” để đổi layout khi vượt qua size đó.
/* Ví dụ media query cơ bản (mobile-first) */
/* Base: mobile */
.container {
  display: flex;
  flex-direction: column;
}

/* Tablet trở lên (>= 768px) */
@media (min-width: 768px) {
  .container {
    flex-direction: row;
  }
}

5.2 Viewport, breakpoints, grid responsive “tay”

Breakpoints phổ biến trong giáo trình WED201c:

Mobile: < 576px
Tablet: 576px - 768px
Small laptop: 768px - 992px
Desktop: ≥ 992px
/* Ví dụ chia 3 cột desktop, xuống mobile thành 1 cột */
.columns { display: flex; flex-direction: column; }
.col { flex: 1; }

@media (min-width: 992px) {
  .columns { flex-direction: row; }
}

5.6 Thực hành responsive & lưu ý khi thi PE

  • ✔ Thiết kế grid 2–3 cột trên desktop, 1 cột trên mobile.
  • ✔ Sử dụng container, row, col-* đúng chuẩn, không dùng table để layout.
  • ✔ Đảm bảo có viewport meta để không bị trừ điểm nặng.
  • ✔ Text không bị quá nhỏ trên mobile, không phải zoom tay.
  • ✔ Menu trên mobile dễ bấm (navbar collapse hoặc menu đơn giản).
